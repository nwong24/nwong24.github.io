### What's the matter with JavaScript
I didn't understand why I had to constrain myself with Typescript at first. Why deal with this typing? Why spend more effort compiling my code? Browsers understand Javascript by default and it was simple and intuitive. With Typescript I had to learn a whole set of types and syntax that was hard to navigate.

### Doing more with less with Typescript
You might be more constrained with Typescript but that constraint can be a source of strength. Imagine working with a team. Without the typing system that Typescript provides, working with pure Javascript can be confusing as you would have to look up the documentation of a certain function to know what types you have to pass in. If you make a mistake it might not even be caught, and could lead to a looong annoying debugging session far into the future. With Typescript, it catches these bugs before you even make the commit!

### What I've learned from Typescript
Learning to work with Typescript has not only made me faster for programming but has also forced me to use better programming practices that I always keep in mind. I'm more disciplined when it comes to type safety, and my code has become more maintainable. Even if I had to switch back to Javascript for a certain team or project, I will be sure to carry these habits along with me.
